1. 자바스크립트?
  ; 웹 페이지에 생동감을 붙여넣기 위해

  - 자바스크립트로 작성한 파일? => Script!
  - 스크립트는 특별한 준비나 컴파일 없이 보통의 문자 형태로 작성 및 실행이 가능
  - 자바스크립트는 자바와는 독자적인 언어이며, 꾸준히 발전을 거듭하면서 ECMAScript라는 고유한 명세를 갖춘 독립적인 언어

1-2. 브라우저에서 하는 일?
 ;  자바스크립트의 능력은 실행 환경에 상당한 영향을 받는다.
 - Node.js 환경에서는 임의의 파일을 읽고, 쓰고, 네트워크 요청을 수행하는 함수를 지원
 - 브라우저 환경에서는 웹페이지 조작, 클라이언트 서버와의 상호작용

1-3. 브라우저에서 할 수 없는 일?
  ; 브라우저는 보안을 위해 자바스크립트의 기능에 제약을 걸어 놓았다.

1-4. 자바스크립트의 강점
  - HTML / CSS 와 완전히 통합할 수 있다.
  - 간단한 일은 간단하게 처리
  - 모든 주요 브라우저에서 지원 및 기본언어로 사용


번외. 자바스크립트 '너머의' 언어들
- TypeScript : 개발을 단순화하고 복잡한 시스템을 지원하려는 목적으로 '자료형의 명시화(strict data typing)' 에 집중해 만든 언어
- Flow : 자료형을 강제, 타입스크립트와는 다른 방식을 사용
- Dart : 모바일 앱과 같이 브라우저가 아닌 환경에서 동작하는 고유의 엔진을 가진 독자적인 언어
